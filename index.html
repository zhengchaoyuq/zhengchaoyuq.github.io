闭包的小知识

换了一个博客，把以前的内容重新写一下格式什么的有时间在弄吧

一点点写吧，主要写一个闭包和promise的小知识

不用闭包的情况

for(var i=0;i<5;i++){
setTimeout(function(){
console.log(i)
},1000i)
}
这个是不是很简单，因为setTimeout会延迟执行，所以会显示5个5，都是想距离1S出现，因为是延迟执行，所以说如果设定的时间是1s的话会
在1s中一同显示出来，但是我们设定的是动态的，所以其实就是按照设定的时间出现，起始的时间都是打印的开始，所以就是都相隔1S而不是
相隔i1000s

使用闭包

for(var i=0;i<5;i++){
(function(i){
setTimeout(function(){
console.log(i)
},1000*i)
})(i)
}
}
在加入闭包以后就会打印出0,1,2,3,4，并且也是间隔1s
那么问题出现了

for(var i=0;i<5;i++){
(function(){
setTimeout(function(){
console.log(i)
},1000*i)
})(i)
}
}

这又会是什么结果呢？？
因为如果把i删掉以后，函数内部并没有调用这个i，所以就会出现和没有使用闭包的情况一样
那么问题又来了

for(var i=0;i<5;i++){
setTimeout((function(i){
console.log(i)
})(i),1000*i)
}

如果函数变成了这种模式又会出现什么样子的情况呢，
按照JS的执行顺序来说的话，自执行函数是立刻执行，然后就自动释放了，但是还有个setTimeout函数延迟执行，那么
setTimeout执行的参数就没有了，就是undefined；所以结果就是直接输出0,1,2,3,4

突然想到了一个考验JS执行顺序的知识
·
(function(){
setTimeout(function(){
console.log(1);
},0)
new Promise(function executor(resolve){
console.log(2);
for(var i=0;i<10000;i++){
i==9999 && resolve();
}
console.log(3);
}).then(function(){
console.log(4);
});
console.log(5)
})()
·
问打印出来的顺序是什么？

重点来了

大家都知道setTimeout是延迟执行，所以第一个肯定打印的不是1,应该是在当前队列；
然后后面是Promise，其中里面的还是是直接执行的，所以先把2,3加入队列中，但是then是回调函数，所以会放在当前序列的最后；
所以先执行5，完后执行4，最后在当前序列完了以后，延迟执行的1才会出现
所以答案是2,3,5,4,1
